# 📌 How to Create Metadata for Markdown Articles

Metadata is a **required section**, placed at the **top of each file** to standardize and manage the documentation system.

---

## 🟣 1. Purpose

* Manage articles consistently
* Search by tags, status, and context
* Track history and content development
* Standardize the entire documentation system

---

## 🔵 2. Standard Structure

Use the `<details>` tag to collapse/expand metadata:

```
<details>
<summary>📌 Meta · ID: GU-001</summary>

<br>

> **Author:** ThuongDev
>
> **Created:** 2026-07-10  
> **Updated:** 2026-07-16
>
> **Context:** Documentation system structure  
> **Status:** Active  
> **Tags:** [#structure](/.hub/nav/tag/cau-truc.vi.md) [#system](/.hub/nav/tag/he-thong.vi.md)  

</details>
```

---

## 🟢 3. Field Explanations

### 3.1 ID

* Format: `XX-000`

  * Examples:

    * `GU-003` → article belongs to **Guide**
    * `TG-002` → article belongs to **Tag**

**Rules:**

* Must be unique and never changed after creation
* Increment sequentially within each prefix group (GU-001 → GU-002 → ...)
* Do not reuse deleted IDs

---

#### 📌 ID Prefix Table

| Prefix | Name       | Meaning        |
| ------ | ---------- | -------------- |
| GU     | Guides     | Guides         |
| TG     | Tag        | Tag navigation |
| ID     | Id         | ID navigation  |
| JO     | Journey    | Journey        |
| DE     | Decisions  | Decisions      |
| SO     | Solutions  | Solutions      |
| ST     | Strategies | Strategies     |

---

### 3.2 Author

* The content creator
* Should follow **PascalCase** format

  * Example: `ThuongDev`

---

### 3.3 Created / Updated

* Standard format: `YYYY-MM-DD`
* `Updated` is only added when there are edits

---

### 3.4 Context

* A brief description of the article context
* Requirements:

  * Concise
  * Clear
  * Easy to understand

---

### 3.5 Status

Recommended statuses:

* `Draft` – In progress, not completed (do not push to GitHub)
* `Active` – Completed and currently in use
* `Deprecated` – No longer suitable, not recommended

---

### 3.6 Tags

* Format: `#tag`

  * Example: `#programming`, `#discrete-math`
* Use link format:

```
[#tag-name](/.hub/nav/tag/tag-name.vi.md)
```

**Rules:**

* Keep concise and consistent
* Use kebab-case (`-`)
* Each article should have 2–4 tags
* Avoid overlapping meanings between tags

**Notes:**

* Articles with IDs starting with `TG`, `ID` **do not need tags**
* See tag list at: `[Tag List]()`

---

## 🔶 4. General Rules

* Metadata must **always be placed at the top of the file**
* Keep formatting consistent across all articles
* Do not modify ID after creation
* Every article must include metadata
