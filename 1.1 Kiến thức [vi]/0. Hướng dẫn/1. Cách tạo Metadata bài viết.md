# 📌 Cách tạo Metadata cho bài viết Markdown

Metadata là **phần bắt buộc**, được đặt ở **đầu mỗi file** nhằm chuẩn hóa và quản lý hệ thống tài liệu.

---

## 🟣 1. Mục đích

* Quản lý bài viết một cách nhất quán
* Tìm kiếm theo tag, trạng thái, ngữ cảnh
* Theo dõi lịch sử và quá trình phát triển nội dung
* Chuẩn hóa toàn bộ hệ thống tài liệu

---

## 🔵 2. Cấu trúc chuẩn

Sử dụng thẻ `<details>` để có thể thu gọn / mở rộng metadata:

```
<details>
<summary>📌 Meta · ID: GU-001</summary>

<br>

> **Tác giả:** ThuongDev
>
> **Ngày tạo:** 2026-07-10  
> **Cập nhật:** 2026-07-16
>
> **Ngữ cảnh:** Cấu trúc hệ thống tài liệu  
> **Trạng thái:** Đang hoạt động  
> **Thẻ:** [#cấu-trúc](/.hub/nav/tag/cau-truc.vi.md) [#hệ-thống](/.hub/nav/tag/he-thong.vi.md)  

</details>
```

---

## 🟢 3. Giải thích từng trường

### 3.1 ID

* Format: `XX-000`
  * Ví dụ:

    * `GU-003` → bài viết thuộc **Guide**
    * `TG-002` → bài viết thuộc **Tag**

**Quy tắc:**

* Không trùng lặp, không thay đổi sau khi tạo
* ID tăng tuần tự theo từng nhóm prefix (GU-001 → GU-002 → ...)
* Không tái sử dụng ID đã xoá

---

#### 📌 Bảng tiền tố ID

| Tiền tố | Tên        | Ý nghĩa             |
| ------- | ---------- | ------------------- |
| GU      | Guides     | Hướng dẫn           |
| TG      | Tag        | Điều hướng theo thẻ |
| ID      | Id         | Điều hướng theo ID  |
| JO      | Journey    | Hành trình          |
| DE      | Decisions  | Quyết định          |
| SO      | Solutions  | Giải pháp           |
| ST      | Strategies | Chiến lược          |

---

### 3.2 Tác giả

* Người tạo nội dung
* Nên thống nhất format **PascalCase**

  * Ví dụ: `ThuongDev`

---

### 3.3 Ngày tạo / Cập nhật

* Định dạng chuẩn: `YYYY-MM-DD`
* `Cập nhật` chỉ thêm khi có chỉnh sửa

---

### 3.4 Ngữ cảnh

* Mô tả ngắn gọn ngữ cảnh bài viết
* Yêu cầu:

  * Ngắn gọn
  * Rõ ràng
  * Dễ hiểu

---

### 3.5 Trạng thái

Các trạng thái khuyến nghị:

* `Bản nháp` – Đang viết, chưa hoàn thiện (không push lên GitHub)
* `Đang hoạt động` – Đã hoàn thiện, đang được sử dụng
* `Đã lỗi thời` - Không còn phù hợp, không khuyến nghị sử dụng

---

### 3.6 Thẻ

* Dạng: `#thẻ`

  * Ví dụ: `#lập-trình`, `#toán-rời-rạc`
* Sử dụng dạng link:

```
[#tên-thẻ](/.hub/nav/tag/ten-the.vi.md)
```

**Quy tắc:**

* Ngắn gọn, thống nhất
* Dùng kebab-case (dấu `-`)
* Không dùng dấu tiếng Việt trong URL
* Mỗi bài nên có 2–4 thẻ
* Tránh trùng nghĩa giữa các thẻ

**Lưu ý:**

* Bài viết có ID bắt đầu bằng `TG`, `ID` **không cần gắn Thẻ**
* Xem danh sách thẻ tại: `[Danh sách thẻ]()`

---

## 🔶 4. Quy tắc chung

* Metadata **luôn đặt ở đầu file**
* Giữ định dạng thống nhất giữa các bài
* Không chỉnh sửa ID sau khi tạo
* Mỗi bài viết bắt buộc phải có metadata

---

## ⚡ 5. Template nhanh

```
<details>
<summary>📌 Meta · ID: XX-000</summary>

<br>

> **Tác giả:** TacGia  
>
> **Ngày tạo:** YYYY-MM-DD  
> **Cập nhật:** YYYY-MM-DD  
>
> **Ngữ cảnh:** Text
> **Trạng thái:** Bản nháp | Đang hoạt động | Đã lỗi thời
> **Thẻ:** #ví-dụ

</details>
```
